The Problem-Solving Mindset
Before you write a single line of code, train your brain
✦ The Golden Rule
An interviewer doesn't care if you get the optimal solution on the first try. They care about how you think. Always narrate your thought process. A wrong answer with clear reasoning beats a correct answer with silence.
📝 Step 1 — Simplify the Problem
Restate the problem in your own words. Remove jargon. Identify: What is the input? What is the output? What is the constraint?

Example: "Given an array, find if any two elements sum to target" → I need to find a pair. I have a list of numbers. Return true/false.
🎯 Step 2 — Spot Target Keywords
Certain words are direct hints. Train your brain to catch them instantly:

frequency / count → HashMap | sorted → Two Pointers / Binary Search | subarray / window → Sliding Window | pair / combination → Two Pointers | shortest path → BFS
🔨 Step 3 — Brute Force First, Always
Never skip brute force. It shows you understand the problem. State the O(n²) or O(2ⁿ) solution first, explain why it's slow, then optimize. This is the most important habit you can build.
⚡ Step 4 — Optimize Incrementally
Ask: "What am I doing repeatedly?" → Can I precompute it? | "What am I searching for?" → Can I use a hash? | "Do I process elements in a fixed window?" → Sliding window. Think in data structures that trade space for time.
📐 Step 5 — Watch the Constraints
Constraints tell you the expected complexity:
n ≤ 10⁸ → O(n) only | n ≤ 10⁶ → O(n log n) ok | n ≤ 10⁴ → O(n²) fine | n ≤ 20 → O(2ⁿ) possible

If n = 10⁵ and you propose O(n²), you'll get a TLE — say it yourself before they do.
🧪 Step 6 — Test with Edge Cases
Always test: empty input, single element, all same values, negative numbers, INT_MIN / INT_MAX, already sorted, reverse sorted. State these to the interviewer before coding.
💡 Interview Tip
When stuck, ask yourself: "If I had unlimited time and memory, how would I solve this?" That's your brute force. Then ask: "What's the bottleneck?" That's where to optimize.

The 5-Step Interview Framework
Follow this every single time — no exceptions
1

Say it back in plain English. Ask: Is input sorted? Can there be duplicates? Are negatives possible? Is space a concern? What to return if no answer? Never assume — always clarify.

3

4

Identify the redundant work. Choose the right data structure. Code the optimal solution. Explain each line as you write. Mention why this works.
5

Dry-run with the given example. Then test your own: empty, single, duplicates, negatives. If you find a bug, fix it calmly — that's fine. Interviewers respect self-correction.

01
Arrays & Strings
The foundation of everything. Master these first.
🧩 Pattern Recognition
Arrays are the default data structure. When no clear pattern emerges, think: Can I sort it? Can I use a prefix sum? Can I traverse in a smarter order?
🧠 Mental Model
Most array problems become easier when you think about indices rather than values. Two pointers, prefix sums, and in-place modifications all revolve around clever index management.

Key question: "Can I avoid an extra loop by maintaining a variable?"
⚡ C++ Arsenal
sort(v.begin(), v.end()) — O(n log n)
reverse(v.begin(), v.end())
rotate(v.begin(), v.begin()+k, v.end())
accumulate(v.begin(), v.end(), 0)
*max_element(v.begin(), v.end())

Key Techniques

Prefix Sum — Range queries in O(1)

Build a prefix sum array, then answer any "sum from index i to j" in O(1).

prefix[i] = prefix[i-1] + arr[i]  |  sum(i,j) = prefix[j] - prefix[i-1]
// Build prefix sum
vector<int> prefix(n+1, 0);
for (int i = 0; i < n; i++)
    prefix[i+1] = prefix[i] + arr[i];

// Query sum of [l, r] (0-indexed)
int rangeSum = prefix[r+1] - prefix[l];
Use when
Multiple range sum queries | "subarray sum equals k" | counting subarrays
In-place Modification — Remove Duplicates, Move Zeros

Use a write pointer that advances only when placing valid elements. The read pointer scans everything.

// Remove duplicates from sorted array
int write = 1;
for (int read = 1; read < n; read++) {
    if (nums[read] != nums[write-1])
        nums[write++] = nums[read];
}
return write;
Trick
Think of write as "the next valid slot". It only moves when you write something valid. Read always moves.
Rotate Array — The Reverse Trick

Rotate right by k: reverse the whole array, then reverse first k, then reverse rest. O(n) time, O(1) space.

void rotate(vector<int>& nums, int k) {
    k %= nums.size(); // handle k > n
    reverse(nums.begin(), nums.end());
    reverse(nums.begin(), nums.begin()+k);
    reverse(nums.begin()+k, nums.end());
}
Product of Array Except Self — No Division Trick

Build prefix product from left and suffix product from right. result[i] = left[i] * right[i].

vector<int> productExceptSelf(vector<int>& nums) {
    int n = nums.size();
    vector<int> res(n, 1);
    int left = 1;
    for (int i = 0; i < n; i++) { res[i] = left; left *= nums[i]; }
    int right = 1;
    for (int i = n-1; i >= 0; i--) { res[i] *= right; right *= nums[i]; }
    return res;
}
02
Two Pointers
Eliminate one loop by being smart about direction
🧩 When to Use
Array is sorted (or can be sorted) | Finding pairs | Palindrome check | Partition problems | Comparing two sequences. Two pointers turns O(n²) brute force into O(n).
↔️ Opposite End Pointers
Start left=0, right=n-1. Move based on condition.

Use for: Two Sum (sorted), Container with Most Water, 3Sum, Valid Palindrome, Trapping Rain Water
→→ Same Direction Pointers
Both start at 0 (slow/fast). Fast scans, slow writes valid elements.

Use for: Remove duplicates, Is Subsequence, Move Zeros, Linked List cycle 
// Two Sum II — sorted array, O(n)
int left = 0, right = nums.size() - 1;
while (left < right) {
    int s = nums[left] + nums[right];
    if (s == target) return {left+1, right+1};
    else if (s < target) left++;   // need bigger
    else right--;                  // need smaller
}
3Sum — Extend Two Pointers to Triplets

Sort first. Fix one element i, then run two pointers on the rest. Skip duplicates carefully.

sort(nums.begin(), nums.end());
for (int i = 0; i < n-2; i++) {
    if (i > 0 && nums[i] == nums[i-1]) continue; // skip dup
    int l = i+1, r = n-1;
    while (l < r) {
        int s = nums[i]+nums[l]+nums[r];
        if (s == 0) { res.push_back({nums[i],nums[l],nums[r]}); l++; r--; }
        else if (s < 0) l++;
        else r--;
    }
}
Key Insight
Sort + fix one element + two pointer = eliminates one loop. Works for 4Sum too (fix two elements).
Container With Most Water — Greedy Two Pointer

Move the shorter wall inward, because moving the taller one can only decrease water.

int left = 0, right = n-1, maxW = 0;
while (left < right) {
    maxW = max(maxW, min(h[left], h[right]) * (right-left));
    if (h[left] < h[right]) left++;
    else right--;
}
⚠️ When NOT to Use Two Pointers
Array is not sorted (and can't be sorted without losing info) → Two pointers rely on monotonicity. Unsorted data means moving a pointer has no predictable effect on the sum/condition.

You need the original indices → Sorting destroys index info. If indices matter (like Two Sum returning indices), use a HashMap instead.

More than 2 variables interact → Two pointers handle one constraint at a time. Multiple constraints need a different approach (e.g. sliding window + map).
✅ Two Pointers Works When
Array is sorted (or sorting it doesn't lose needed info) | You're searching for a pair or triplet satisfying a condition | Problem involves opposite ends shrinking toward each other | Elements are compared or combined and direction of pointer movement is deterministic.
03
Sliding Window
Process subarrays / substrings without recomputing from scratch
🧩 When to Use
Contiguous subarray / substring questions | "maximum/minimum in a window of size k" | "longest substring with condition" | "minimum window containing". Sliding window = O(n) instead of O(n²).
📏 Fixed Window
Window size k is given. Add new element, remove old element. Maintain running state.

Use for: Max sum of subarray of size k, average of window
🔄 Variable Window
Expand right until condition violated. Then shrink left until valid again.

Use for: Longest substring without repeating, Minimum window substring, Minimum size subarray sum 
// Variable window template — longest substring without repeating
unordered_set<char> window;
int left = 0, maxLen = 0;
for (int right = 0; right < s.size(); right++) {
    while (window.count(s[right]))  // shrink if duplicate
        window.erase(s[left++]);
    window.insert(s[right]);
    maxLen = max(maxLen, right - left + 1);
}
Minimum Window Substring — HashMap Sliding Window

Count required chars. Expand right, track "have". When have==need, try shrinking left for minimum.

unordered_map<char,int> need, window;
for (char c : t) need[c]++;
int have = 0, required = need.size(), l = 0, minLen = INT_MAX, start = 0;
for (int r = 0; r < s.size(); r++) {
    window[s[r]]++;
    if (need.count(s[r]) && window[s[r]] == need[s[r]]) have++;
    while (have == required) {
        if (r-l+1 < minLen) { minLen = r-l+1; start = l; }
        window[s[l]]--;
        if (need.count(s[l]) && window[s[l]] < need[s[l]]) have--;
        l++;
    }
}
Mental Model
Think of it as a stretchy rubber band — expand until you have what you need, then squeeze from the left to minimize.
⚠️ When NOT to Use Sliding Window
Negative numbers in array → Sliding window breaks because shrinking the window doesn't guarantee the sum decreases. Use prefix sum + hashmap instead (e.g. Subarray Sum = K).

Non-contiguous elements needed → Sliding window only works on contiguous subarrays/substrings. For subsequences, use DP or two pointers.

Condition depends on order of elements → If validity depends on something other than a simple count or sum (e.g. sorted order inside window), sliding window won't apply cleanly.

Multiple disjoint windows needed → Sliding window handles one window at a time. For multiple simultaneous ranges, think prefix sum or segment trees.
✅ Sliding Window Works When
All elements are non-negative (adding to window always increases, removing always decreases) | Problem asks for contiguous subarray or substring | There's a clear expand/shrink condition | Window validity is monotonic — if window of size k is invalid, size k+1 is also invalid (or vice versa).

Hashing
O(1) lookup — the swiss army knife of optimization
🧩 When to Use
Any time you need to count, check existence, or group items. HashMap makes O(n²) brute force → O(n). If you're doing a nested loop to find "complement" or "match", there's likely a hashmap solution.
# Count / Frequency
unordered_map<T, int> freq;
Then freq[x]++ for each element. Used in anagrams, ransom note, group anagrams.
✓ Existence Check
unordered_set<T> seen;
Check seen.count(x), add with seen.insert(x). Used in Two Sum, Longest Consecutive.
🔗 Value → Index Mapping
unordered_map<int, int> idx;
Store idx[val] = i. Lookup idx[target-val]. Classic Two Sum trick.
📦 Grouping
map<string, vector<string>> groups;
Sort each string as key → group anagrams. Or use frequency signature as key.
// Two Sum — O(n) with hashmap
unordered_map<int,int> idx;
for (int i = 0; i < n; i++) {
    int complement = target - nums[i];
    if (idx.count(complement))
        return {idx[complement], i};
    idx[nums[i]] = i;
}
Longest Consecutive Sequence — HashSet Magic

Add all elements to a set. For each element, only start counting if x-1 is NOT in set (it's a sequence start). Then count up.

unordered_set<int> s(nums.begin(), nums.end());
int maxLen = 0;
for (int x : s) {
    if (!s.count(x-1)) { // start of sequence
        int len = 1;
        while (s.count(x+len)) len++;
        maxLen = max(maxLen, len);
    }
}
// O(n) — each element visited at most twice
Key Insight
By only starting at sequence beginnings, each element is processed at most twice total → O(n).
Group Anagrams — Sorted String as Key 
unordered_map<string, vector<string>> groups;
for (auto& w : words) {
    string key = w;
    sort(key.begin(), key.end());
    groups[key].push_back(w);
}
Alternative Key
Use a frequency array as key: "a2b1c3" — avoids O(k log k) sorting cost per word.
⚠️ HashMap Gotchas in C++
unordered_map vs map → unordered_map is O(1) average but O(n) worst case (hash collision). Use map (O(log n)) when you need sorted order or keys are custom types.

Accessing a missing key creates it → mp[key] inserts a default 0 if key doesn't exist. Use mp.count(key) or mp.find(key) to check existence without inserting.

Floating point keys → Never use float/double as hashmap keys. Precision errors make equal values hash differently. Multiply and use integers instead.

Erasing while iterating → Don't erase from a map while ranging over it with a range-for. Use index-based loop or collect keys to erase separately.

Matrix / 2D Arrays
Think in layers — rotate, spiral, or scan with direction arrays
🧩 Mental Model
Matrix problems usually require direction arrays or layer-by-layer processing. Always think: "Can I do this in-place?" and "What happens at boundaries?"
💡 Direction Array Template
For 4-directional traversal (used in islands, BFS, Game of Life):
int dx[] = {0,0,1,-1}; int dy[] = {1,-1,0,0};
Then loop: int nx=x+dx[i], ny=y+dy[i]; if(valid bounds) process
Spiral Matrix — Boundary Shrinking

Maintain 4 boundaries: top, bottom, left, right. Traverse edges, then shrink boundaries inward.

int top=0, bot=m-1, left=0, right=n-1;
while (top <= bot && left <= right) {
    for(int i=left; i<=right; i++) res.push_back(mat[top][i]); top++;
    for(int i=top; i<=bot; i++) res.push_back(mat[i][right]); right--;
    if(top<=bot) { for(int i=right; i>=left; i--) res.push_back(mat[bot][i]); bot--; }
    if(left<=right) { for(int i=bot; i>=top; i--) res.push_back(mat[i][left]); left++; }
}
Rotate Image 90° — Transpose + Reverse

Two steps: 1) Transpose (swap [i][j] with [j][i]), 2) Reverse each row.

// Transpose
for(int i=0; i<n; i++)
    for(int j=i+1; j<n; j++)
        swap(mat[i][j], mat[j][i]);
// Reverse each row
for(int i=0; i<n; i++)
    reverse(mat[i].begin(), mat[i].end());
Pattern
Rotate 90° CW = Transpose + Reverse rows. Rotate 90° CCW = Transpose + Reverse columns. Rotate 180° = Reverse entire matrix.
06
Kadane's Algorithm
Maximum subarray in O(n) — a DP in disguise
✦ Core Insight
At each position i, the maximum subarray ending at i is either: (1) just nums[i], or (2) the best subarray ending at i-1 extended by nums[i]. If previous best is negative, restart from current element.
dp[i] = max(nums[i], dp[i-1] + nums[i])
int maxSum = nums[0], curSum = nums[0];
for (int i = 1; i < n; i++) {
    curSum = max(nums[i], curSum + nums[i]);
    maxSum = max(maxSum, curSum);
}
// O(n) time, O(1) space
Extension
Maximum Sum Circular Subarray: Answer is either (a) max subarray of original array, or (b) total_sum − min subarray (take the wrap-around part). Use Kadane twice!

Binary Search
If the search space is monotonic, binary search it
🧩 The Expanded Mental Model
Binary search is not just for sorted arrays. Use it whenever: the answer space is monotonic (if X works, X-1 also works), or when you're searching in a sorted structure, or looking for a peak/boundary.
✦ Template — Never Get Off-by-One Again
Use while(lo <= hi) for exact match search. Use while(lo < hi) for boundary finding. Always think: "What happens when the loop exits? What do lo and hi point to?" 
// Standard binary search — O(log n)
int lo = 0, hi = n - 1;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2; // avoids overflow vs (lo+hi)/2
    if (nums[mid] == target) return mid;
    else if (nums[mid] < target) lo = mid + 1;
    else hi = mid - 1;
}
return -1;
Search in Rotated Sorted Array

One half is always sorted. Check which half, then decide which half to search.

while (lo <= hi) {
    int mid = lo + (hi-lo)/2;
    if (nums[mid] == target) return mid;
    if (nums[lo] <= nums[mid]) { // left half sorted
        if (target >= nums[lo] && target < nums[mid]) hi = mid-1;
        else lo = mid+1;
    } else { // right half sorted
        if (target > nums[mid] && target <= nums[hi]) lo = mid+1;
        else hi = mid-1;
    }
}
Find Peak Element — Binary on Answer 
int lo = 0, hi = n-1;
while (lo < hi) {
    int mid = lo + (hi-lo)/2;
    if (nums[mid] > nums[mid+1]) hi = mid;  // peak is left or mid
    else lo = mid+1;                         // peak is right
}
return lo;
⚠️ Classic Bug
Always compute mid as lo + (hi - lo) / 2, not (lo + hi) / 2. The latter overflows when lo and hi are large integers!
⚠️ When Binary Search Fails
Array is not sorted → Binary search requires sorted (or rotated-sorted) data. On unsorted data the result is undefined.

Duplicate elements + need exact index → Standard binary search returns any matching index. For first/last occurrence, use lower_bound / upper_bound.

Answer space is not monotonic → Binary-search-on-answer only works when the function is monotonic: if X works, X+1 also works (or doesn't). If there are valleys and peaks in the answer space, binary search gives wrong results.

Infinite loop risk → If you write lo = mid instead of lo = mid + 1, you can get stuck. Always make sure lo and hi strictly converge each iteration.
💡 lower_bound / upper_bound — Built-in C++ Binary Search
lower_bound(v.begin(), v.end(), x) → iterator to first element ≥ x
upper_bound(v.begin(), v.end(), x) → iterator to first element > x
Subtract v.begin() to get index. Both are O(log n) on sorted vectors.

Stacks & Queues
LIFO and FIFO — the structural backbone of expression problems
📚 Stack Use Cases
  • Matching brackets / parentheses
  • Monotonic stack (next greater element)
  • Expression evaluation (RPN)
  • Undo/redo operations
  • DFS iterative
  • Simplify file paths
🔁 Queue Use Cases
  • BFS traversal
  • Level-order tree traversal
  • Sliding window maximum (deque)
  • Task scheduling
  • LRU Cache (deque + map)
// Valid Parentheses
stack<char> st;
for (char c : s) {
    if (c=='('||c=='{'||c=='[') st.push(c);
    else {
        if (st.empty()) return false;
        char top = st.top(); st.pop();
        if ((c==')'&&top!='(')||(c=='}'&&top!='{')||(c==']'&&top!='[')) return false;
    }
}
return st.empty();
Min Stack — Track Minimum with O(1) getMin()

Use a second stack that only pushes when a new minimum is found.

stack<int> st, minSt;
void push(int val) {
    st.push(val);
    if (minSt.empty() || val <= minSt.top()) minSt.push(val);
}
void pop() {
    if (st.top() == minSt.top()) minSt.pop();
    st.pop();
}
int getMin() { return minSt.top(); }
09
Linked Lists
Pointer manipulation — draw it out, always
🧩 The Golden Rule
Before touching any linked list problem: draw the nodes on paper. Label your pointers. Pointer bugs are invisible in your head but obvious on paper. Use a dummy node to avoid head-edge-case bugs.
🐢🐇 Fast & Slow Pointers
Fast moves 2 steps, slow moves 1. When fast hits end, slow is at middle.

Use for: Detect cycle, find middle, find nth from end
🪄 Dummy Node
Create a dummy head: ListNode* dummy = new ListNode(0); dummy->next = head;

Eliminates special cases for inserting at head or empty list. Always return dummy->next. 
// Reverse Linked List — iterative
ListNode* prev = nullptr, *curr = head;
while (curr) {
    ListNode* nxt = curr->next;
    curr->next = prev;
    prev = curr;
    curr = nxt;
}
return prev;
Floyd's Cycle Detection 
ListNode* slow = head, *fast = head;
while (fast && fast->next) {
    slow = slow->next;
    fast = fast->next->next;
    if (slow == fast) return true;
}
return false;
Finding Cycle Start
After detection, reset slow to head. Move both one step at a time. Where they meet = cycle start. Mathematical proof: distance from head to cycle start = distance from meeting point to cycle start.
Remove Nth Node From End

Use two pointers separated by n+1 gap. When fast hits null, slow is at the node before the target.

ListNode* dummy = new ListNode(0, head);
ListNode* fast = dummy, *slow = dummy;
for(int i=0; i<=n; i++) fast = fast->next;
while(fast) { slow=slow->next; fast=fast->next; }
slow->next = slow->next->next;
return dummy->next;

Trees & BST
Recursion is your superpower — trust the call stack
✦ The Recursive Leap of Faith
Assume your function works correctly for all subtrees. Then use it for the root. This "black box" thinking is the key to tree recursion. Define: what does this function return? Be precise. Never deviate from that contract.
🌿 DFS Traversals
Preorder (root, left, right) — copy tree, serialize
Inorder (left, root, right) — BST gives sorted order
Postorder (left, right, root) — delete tree, check subtrees first
🌊 BFS / Level Order
Use a queue. Process all nodes at depth d before depth d+1.

Use for: Right side view, average of levels, zigzag traversal, minimum depth 
// BFS Level Order — queue template
queue<TreeNode*> q;
q.push(root);
while (!q.empty()) {
    int sz = q.size(); // snapshot this level's size
    for (int i = 0; i < sz; i++) {
        TreeNode* node = q.front(); q.pop();
        // process node
        if (node->left) q.push(node->left);
        if (node->right) q.push(node->right);

Validate BST — Pass Bounds Down 
bool validate(TreeNode* node, long lo, long hi) {
    if (!node) return true;
    if (node->val <= lo || node->val >= hi) return false;
    return validate(node->left, lo, node->val)
        && validate(node->right, node->val, hi);
}
// Call: validate(root, LONG_MIN, LONG_MAX)
Common Mistake
Checking only node->left->val < node->val is WRONG. The entire left subtree must be less. Pass bounds down recursively.
Lowest Common Ancestor 
TreeNode* lca(TreeNode* root, TreeNode* p, TreeNode* q) {
    if (!root || root==p || root==q) return root;
    TreeNode* left  = lca(root->left, p, q);
    TreeNode* right = lca(root->right, p, q);
    if (left && right) return root; // p and q on different sides
    return left ? left : right;
}

Dynamic Programming
Break into subproblems. Remember answers. Build bottom-up.
✦ The DP Mindset — 3 Questions
1. What is the state? (What changes at each step?)
2. What is the transition? (How does state[i] depend on previous states?)
3. What is the base case? (Where does the recursion bottom out?)
⚠️ When DP is NOT the Right Approach
No overlapping subproblems → If every subproblem is unique (like most backtracking problems without repeated states), memoization adds overhead with no benefit.

Optimal substructure doesn't hold → If the optimal solution to the whole problem doesn't include optimal solutions to subproblems (e.g. some shortest path problems with constraints), DP gives wrong answers.

State space is too large → If dp[i][j][k] with large ranges, memory explodes. Check if space optimization (rolling array) is possible or if greedy works instead.
🧩 Greedy vs DP — How to Decide
Try greedy first — if a locally optimal choice always leads to a global optimum, greedy works and is faster.
Use DP when greedy fails — when a choice now affects future choices in a non-trivial way, you need to explore all options and remember results.

Quick test: Can you prove greedy works with an exchange argument? If yes → greedy. If the problem says "number of ways" or "minimum over all paths" → DP.
⬇️ Top-Down (Memoization)
Recursive function + memo map/array. Natural to write, easier to reason about. Good for sparse problems where not all states are visited. 
unordered_map<int,int> memo;
int dp(int n) {
    if(n<=1) return n;
    if(memo.count(n)) return memo[n];
    return memo[n] = dp(n-1) + dp(n-2);
}
⬆️ Bottom-Up (Tabulation)
Fill a table iteratively from base cases. Better cache performance, no recursion stack overflow risk. 
vector<int> dp(n+1);
dp[0]=1; dp[1]=1;
for(int i=2; i<=n; i++)
    dp[i] = dp[i-1] + dp[i-2];

Intervals
Sort first. Then scan greedily.
🧩 The Template
Step 1: Sort by start time. Step 2: Initialize current interval. Step 3: For each next interval — if overlap, merge (extend end). If no overlap, add current to result, update current. 
// Merge Intervals
sort(intervals.begin(), intervals.end());
vector<vector<int>> res = {intervals[0]};
for(int i=1; i<intervals.size(); i++) {
    if(intervals[i][0] <= res.back()[1])
        res.back()[1] = max(res.back()[1], intervals[i][1]);
    else
        res.push_back(intervals[i]);
}
return res;
Overlap Condition
Two intervals [a,b] and [c,d] overlap if and only if c ≤ b (next start ≤ current end). They DON'T overlap if c > b. Merge by taking max of ends.

The mental playbook for upgrading every naive solution
✦ The Optimization Thought Process
Every brute force has a bottleneck. Ask yourself:
"What am I recomputing on every iteration?" → Precompute it (prefix sum, DP)
"What am I searching for inside a loop?" → Replace with HashMap or Set O(1) lookup
"Am I scanning the whole array every step?" → Try sliding window or two pointers
"Am I trying all combinations?" → Try sorting + greedy, or DP

Trigger: "Find two elements that satisfy a condition (sum, difference, match)"
Brute: Try all pairs with i,j loops → O(n²)
Insight: For element x, you need y = target - x. Store seen elements in a map. Look up instead of loop.
Result: O(n) time, O(n) space

Monotonic Stack
The underrated weapon for "next greater / smaller" problems in O(n)
🧩 When to Use
"Next greater element", "previous smaller element", "largest rectangle in histogram", "stock span", "circular array NGE". A monotonic stack processes each element once → O(n) instead of O(n²).
📈 Monotonic Increasing Stack
Stack bottom → top goes increasing. Pop when current element is smaller than top.

Use for: Previous greater element, stock span
while(!st.empty() && st.top() >= curr) st.pop();
📉 Monotonic Decreasing Stack
Stack bottom → top goes decreasing. Pop when current element is larger than top.

Use for: Next greater element, circular NGE
while(!st.empty() && st.top() <= curr) st.pop();
Next Greater Element — Monotonic Decreasing Stack 
vector<int> nextGreater(vector<int>& arr) {
  int n = arr.size();
  vector<int> ans(n, -1);
  stack<int> st; // stores indices
  for(int i = 0; i < n; i++) {
    while(!st.empty() && arr[st.top()] < arr[i]) {
      ans[st.top()] = arr[i]; // found NGE for stack top
      st.pop();
    }
    st.push(i);
  }
  return ans; // remaining in stack have no NGE → stays -1
}
Pattern
Store indices not values in the stack — you need the index to update the answer array.
Largest Rectangle in Histogram — O(n)

Use monotonic increasing stack of indices. When a shorter bar appears, pop and calculate area.

int largestRectangle(vector<int>& heights) {
  int maxA = 0, n = heights.size();
  stack<int> st;
  for(int i = 0; i <= n; i++) {
    int h = (i == n) ? 0 : heights[i];
    while(!st.empty() && heights[st.top()] >= h) {
      int height = heights[st.top()]; st.pop();
      int width = st.empty() ? i : i - st.top() - 1;
      maxA = max(maxA, height * width);
    }
    st.push(i);
  }
  return maxA;
}
Width Trick
Width = i - stack.top() - 1. If stack is empty after pop, width = i (spans all the way to start).
Sliding Window Maximum — Deque (Monotonic Decreasing)

Maintain a deque where front is always the index of the current window's max. Remove out-of-window indices from front. Remove smaller elements from back.

vector<int> maxSlidingWindow(vector<int>& nums, int k) {
  deque<int> dq; // stores indices
  vector<int> res;
  for(int i = 0; i < nums.size(); i++) {
    while(!dq.empty() && dq.front() < i-k+1) dq.pop_front(); // out of window
    while(!dq.empty() && nums[dq.back()] < nums[i]) dq.pop_back(); // smaller useless
    dq.push_back(i);
    if(i >= k-1) res.push_back(nums[dq.front()]);
  }
  return res;
} // O(n) — each element added/removed from deque at most once
Asteroid Collision — Stack Simulation

Positive = moves right, Negative = moves left. Only collide when top of stack is positive and current is negative.

vector<int> asteroidCollision(vector<int>& a) {
  stack<int> st;
  for(int x : a) {
    bool alive = true;
    while(alive && x < 0 && !st.empty() && st.top() > 0) {
      if(st.top() < -x) { st.pop(); continue; }         // right smaller, explodes
      else if(st.top() == -x) { st.pop(); }               // equal, both explode
      alive = false;                                        // left smaller or equal, dies
    }
    if(alive) st.push(x);
  }
  vector<int> res; while(!st.empty()) { res.push_back(st.top()); st.pop(); }
  reverse(res.begin(), res.end());
  return res;
}

Dijkstra's Shortest Path — Min-Heap Template 
vector<int> dijkstra(int n, vector<vector<pair<int,int>>>& adj, int src) {
  vector<int> dist(n, 1e9); dist[src]=0;
  priority_queue<pair<int,int>, vector<pair<int,int>>, greater<>> pq;
  pq.push({0,src});
  while(!pq.empty()) {
    auto [d,u] = pq.top(); pq.pop();
    if(d > dist[u]) continue; // stale entry
    for(auto [v,w] : adj[u])
      if(dist[u]+w < dist[v]) { dist[v]=dist[u]+w; pq.push({dist[v],v}); }
  }
  return dist;
} // O((V+E) log V)
Stale Entry Check
Always check if(d > dist[u]) continue; — old (outdated) distance entries may be in the heap. Skipping them is what makes Dijkstra correct.

Backtracking — Deep Dive
Explore all possibilities, prune dead ends, backtrack and try again
✦ The Backtracking Mental Model
Backtracking = DFS on a decision tree.
At each node: make a choice → recurse → undo the choice.
The "undo" step is what makes it backtracking — not just recursion.

Ask yourself: What choices do I have at this step? What's the base case (when am I done)? What makes a choice invalid (prune early)?
🌿 Universal Backtracking Template
Every backtracking problem follows this exact skeleton. Only the base case, choices, and validity check change.
✂️ Pruning = Speed
The power of backtracking isn't brute force — it's early pruning. Skip invalid branches before recursing. Sorting the input often enables the best pruning. 
// Universal Backtracking Template — memorize this
vector<vector<int>> result;
vector<int> path;

void backtrack(vector<int>& nums, int start) {
  // ① Base case — goal reached
  if (goal_reached) {
    result.push_back(path);
    return;
  }

  // ② Explore all choices
  for (int i = start; i < nums.size(); i++) {
    // ③ Prune invalid choices early
    if (not_valid(nums[i])) continue;

    // ④ Make choice
    path.push_back(nums[i]);

    // ⑤ Recurse
    backtrack(nums, i + 1); // or i for reuse

    // ⑥ Undo choice (backtrack)
    path.pop_back();
  }
}
⚠️ Common Mistake
Forgetting the pop_back() (undo step). Without it, path keeps growing and gives wrong results. The undo always mirrors the make step perfectly.

Subsets (LC 78) — Add at every node, not just leaves
Key Insight
Unlike combination/permutation problems, we add to result at every recursive call (including intermediate nodes), not only at leaf nodes. Every path from root represents a valid subset.
Brute
Bitmask: 2ⁿ iterations, check each bit
O(n × 2ⁿ) — readable but verbose
→
Optimal
Backtracking — pick/skip at each index
O(n × 2ⁿ) — cleaner and extensible
vector<vector<int>> res;

void backtrack(vector<int>& nums, int start, vector<int>& path) {
  res.push_back(path);            // add current subset (every node is valid)
  for (int i = start; i < nums.size(); i++) {
    path.push_back(nums[i]);       // pick nums[i]
    backtrack(nums, i + 1, path); // explore subsets starting after i
    path.pop_back();               // undo pick (backtrack)
  }
}

vector<vector<int>> subsets(vector<int>& nums) {
  vector<int> path;
  backtrack(nums, 0, path);
  return res; // 2^n subsets total
}

Subsets II (LC 90) — Handle duplicates by sorting + skipping
Key Insight
Sort first so duplicates are adjacent. At the same recursion level (same start), skip if nums[i] == nums[i-1]. This prevents generating the same subset twice from different starting positions.
void backtrack(vector<int>& nums, int start, vector<int>& path) {
  res.push_back(path);
  for (int i = start; i < nums.size(); i++) {
    // Skip duplicate at the SAME level (i > start, not i > 0)
    if (i > start && nums[i] == nums[i-1]) continue;
    path.push_back(nums[i]);
    backtrack(nums, i + 1, path);
    path.pop_back();
  }
}

vector<vector<int>> subsetsWithDup(vector<int>& nums) {
  sort(nums.begin(), nums.end()); // MUST sort first
  vector<int> path;
  backtrack(nums, 0, path);
  return res;
}
i > start vs i > 0
i > start skips duplicates at the same recursion depth (siblings). i > 0 would also skip across depths — that's wrong and would miss valid subsets like [2,2] from [1,2,2].
Permutations (LC 46) — All orderings using swap trick
Key Insight
At position start, every element from start to n-1 can go here. Swap it into position, recurse for rest, then swap back. No extra visited array needed.
void backtrack(vector<int>& nums, int start) {
  if (start == nums.size()) { res.push_back(nums); return; } // leaf node
  for (int i = start; i < nums.size(); i++) {
    swap(nums[start], nums[i]); // place nums[i] at position start
    backtrack(nums, start + 1);
    swap(nums[start], nums[i]); // restore original order (backtrack)
  }
}

vector<vector<int>> permute(vector<int>& nums) {
  backtrack(nums, 0);
  return res; // n! permutations
}
Alternative — visited array approach
Instead of swap, use a bool visited[n] array. For each position, pick any unvisited element, mark visited, recurse, then unmark. Both approaches give O(n × n!) but swap is O(1) space. 
// Alternative: visited array (easier to understand)
vector<bool> vis(nums.size(), false);
void backtrack(vector<int>& nums, vector<int>& path) {
  if (path.size() == nums.size()) { res.push_back(path); return; }
  for (int i = 0; i < nums.size(); i++) {
    if (vis[i]) continue;
    vis[i] = true;
    path.push_back(nums[i]);
    backtrack(nums, path);
    path.pop_back();
    vis[i] = false;
  }
}
Combination Sum (LC 39) — Reuse elements, prune by target
Key Insight
Each element can be used unlimited times. Pass i (not i+1) to the recursive call so the same element can be picked again. Prune by checking if nums[i] > target — no need to go further if already exceeding.
void backtrack(vector<int>& nums, int target, int start, vector<int>& path) {
  if (target == 0) { res.push_back(path); return; } // exact match
  for (int i = start; i < nums.size(); i++) {
    if (nums[i] > target) break; // sorted → no point continuing
    path.push_back(nums[i]);
    backtrack(nums, target - nums[i], i, path); // i (not i+1) → reuse allowed
    path.pop_back();
  }
}

vector<vector<int>> combinationSum(vector<int>& nums, int target) {
  sort(nums.begin(), nums.end()); // sort enables pruning
  vector<int> path;
  backtrack(nums, target, 0, path);
  return res;
}
i vs i+1 — The Critical Difference
Pass i → element can be reused (Combination Sum I)
Pass i+1 → each element used at most once (Combination Sum II, Subsets)
Pass 0 → all elements available at every step (Permutations)
Combination Sum II (LC 40) — Each element once + skip duplicates
Key Insight
Each number used once, but array has duplicates. Sort first, then skip duplicates at the same level (i > start && nums[i] == nums[i-1]) — same trick as Subsets II.
void backtrack(vector<int>& nums, int target, int start, vector<int>& path) {
  if (target == 0) { res.push_back(path); return; }
  for (int i = start; i < nums.size(); i++) {
    if (i > start && nums[i] == nums[i-1]) continue; // skip dup at same level
    if (nums[i] > target) break;                       // prune
    path.push_back(nums[i]);
    backtrack(nums, target - nums[i], i + 1, path);   // i+1 → no reuse
    path.pop_back();
  }
}

Combinations (LC 77) — Choose k elements from 1..n
Key Insight
Classic subset pattern with a size constraint. Stop recursing when path.size() == k. Add an extra pruning: if remaining elements aren't enough to fill k slots, prune early.
void backtrack(int n, int k, int start, vector<int>& path) {
  if (path.size() == k) { res.push_back(path); return; }

  // Pruning: need (k - path.size()) more elements, only (n - i + 1) available
  for (int i = start; i <= n - (k - (int)path.size()) + 1; i++) {
    path.push_back(i);
    backtrack(n, k, i + 1, path);
    path.pop_back();
  }
}

vector<vector<int>> combine(int n, int k) {
  vector<int> path;
  backtrack(n, k, 1, path);
  return res;
}
Pruning formula
Loop only up to n - (k - path.size()) + 1. If you need 2 more elements and only 1 is left, there's no point trying — prune it.

Generate Parentheses (LC 22) — Count-based pruning
Key Insight
Two rules define validity:
1. Can add ( only if open count < n
2. Can add ) only if close count < open count
These two rules prune the entire invalid search space. 
void backtrack(int n, int open, int close, string& path) {
  if (path.size() == 2*n) { res.push_back(path); return; }

  if (open < n) {              // can still add '('
    path.push_back('(');
    backtrack(n, open+1, close, path);
    path.pop_back();
  }
  if (close < open) {          // can close only if more opens exist
    path.push_back(')');
    backtrack(n, open, close+1, path);
    path.pop_back();
  }
}

vector<string> generateParenthesis(int n) {
  string path;
  backtrack(n, 0, 0, path);
  return res; // Catalan(n) total valid strings
}
No explicit undo needed?
We still do pop_back() — that IS the undo. Passing open/close as values (not references) means the counts auto-reset. Passing path by reference requires explicit pop_back.
Word Search (LC 79) — Grid DFS with visited marking
Key Insight
Mark cell as visited by temporarily replacing it with '#'. After recursion, restore original character — that's the backtrack step. No separate visited array needed.
bool dfs(vector<vector<char>>& board, string& word, int i, int j, int k) {
  if (k == word.size()) return true;  // matched all characters
  if (i<0 || j<0 || i>=board.size() || j>=board[0].size()) return false;
  if (board[i][j] != word[k]) return false; // mismatch

  char tmp = board[i][j];
  board[i][j] = '#';            // mark visited
  bool found = dfs(board,word,i+1,j,k+1) || dfs(board,word,i-1,j,k+1)
             || dfs(board,word,i,j+1,k+1) || dfs(board,word,i,j-1,k+1);
  board[i][j] = tmp;             // restore (backtrack)
  return found;
}

bool exist(vector<vector<char>>& board, string word) {
  for (int i=0; i<board.size(); i++)
    for (int j=0; j<board[0].size(); j++)
      if (dfs(board, word, i, j, 0)) return true;
  return false;
}
N-Queens (LC 51) — Row-by-row placement with column/diagonal tracking
Key Insight
Place one queen per row. Track occupied columns and both diagonals using sets. For row r, col c: main diagonal = r-c, anti-diagonal = r+c. These are unique per diagonal.
set<int> cols, diag1, diag2; // occupied columns, r-c, r+c
vector<string> board;

void backtrack(int n, int row) {
  if (row == n) { res.push_back(board); return; }
  for (int col = 0; col < n; col++) {
    if (cols.count(col) || diag1.count(row-col) || diag2.count(row+col))
      continue; // conflict — skip this column

    // Place queen
    cols.insert(col); diag1.insert(row-col); diag2.insert(row+col);
    string rowStr(n, '.'); rowStr[col] = 'Q';
    board.push_back(rowStr);

    backtrack(n, row + 1);

    // Remove queen (backtrack)
    cols.erase(col); diag1.erase(row-col); diag2.erase(row+col);
    board.pop_back();
  }
}

vector<vector<string>> solveNQueens(int n) {
  backtrack(n, 0);
  return res;
}
Diagonal Math
All cells on the same top-left→bottom-right diagonal share the same row - col value. All cells on the same top-right→bottom-left diagonal share the same row + col value. Use these as keys in sets for O(1) conflict checks.
Palindrome Partitioning (LC 131) — Partition + validate
Key Insight
At each position, try every possible substring starting from start. If it's a palindrome, add it to path and recurse from the next index. When start reaches end of string, all partitions are palindromes — record it.
bool isPalin(string& s, int l, int r) {

Sudoku Solver (LC 37) — Constraint propagation + backtrack 
bool isValid(vector<vector<char>>& board, int row, int col, char num) {
  for (int i=0; i<9; i++) {
    if (board[row][i] == num) return false;  // row conflict
    if (board[i][col] == num) return false;  // col conflict
    if (board[3*(row/3) + i/3][3*(col/3) + i%3] == num) return false; // box
  }
  return true;
}

bool solve(vector<vector<char>>& board) {
  for (int r=0; r<9; r++) {
    for (int c=0; c<9; c++) {
      if (board[r][c] != '.') continue;
      for (char ch='1'; ch<='9'; ch++) {
        if (isValid(board, r, c, ch)) {
          board[r][c] = ch;           // place digit
          if (solve(board)) return true;
          board[r][c] = '.';         // backtrack
        }
      }
      return false; // no valid digit → backtrack to previous cell
    }
  }
  return true; // all cells filled
}
Box index trick
3*(row/3) + i/3 and 3*(col/3) + i%3 iterates all 9 cells of the 3×3 box containing (row, col). Memorize this formula.

🔥 Backtracking in Coding Rounds — Speed Tips
1. Sort input first — enables early pruning with if(nums[i] > target) break;

3. Pass path by reference + push/pop instead of copying strings/vectors each call

6. Return early with true (short-circuit) when you just need existence, not all solutions
